ICCT health tool

This tool is designed to be used in combination with the Port Emissions Tool for rapid assessment of the health impact from port emissions. See details below, or click continue to access the tool.

Model

This is part 3 of a 3-phase project. The first phase is the Port Emissions Tool, where port emissions are estimated from port activities. The second phase is building the emissions to concentrations matrix for 6 specific ports, including Aqaba, Chittagong, Jakarta, Shenzhen, Tema, Valparaiso, and four generalized ports (see Appendix for details). The four general ports face four different directions, and are designed to be used as any port that faces the same direction. The domain is 100km * 100km. The third phase is assessing the health impact of port emissions using the matrix developed in the second step. The algorithm of this tool is based on ICCT health model. See details here . Pollutants include fine primary PM2.5, secondary sulfates and secondary nitrates. Diseases include acute respiratory infections (ARI), lung cancer (LC) and cardiopulmonary (CP) diseases. This tool is designed for rapid assessment only. Each run of the model only calculates the health impact from that specific year’s port emissions and the results are not lagged. Results are not saved to a database.

Data

Population data in the domains: Gridded Population of the World, Version 3 (GPWv3) Data Collection 1

Population growth rate and structure: United Nations, Department of Economic and Social Affairs, Population Division (2015). World Population Prospects: The 2015 Revision

Incidence rate: GBD 2013 data

Ambient concentration: Global Annual Average PM2.5 Grids from MODIS and MISR Aerosol Optical Depth (AOD).2,3

Inputs

Required Inputs

These are the inputs that are necessary for the model to calculate the health impact. The model either cannot choose them for the user, or does not have such data itself. These fields are now pre-filled with dummy data. Please change them to real data. (specify which field of the PEIT output corresponds to what input here….)

Port: Choose specific port if it applies, otherwise choose the general port. The user will also need to choose a direction that the port is facing when choosing to use a generalized port. For example, if a port has open water on the northern part and land on the southern part, its direction is north.

Modeled year: 2001 to 2014. Due to data availability, the model does not have ambient PM2.5 concentration data for years after 2010. Ambient concentration of 2010 will be used instead for years after 2010. Users can input thier own concentration data in the ‘optional inputs’ page.

Total annual PM10 emissions: This is used to estimate the concentration of PM10, which is required to calculate deaths from ARI.

Direction: Only need to input the direction when choosing a general port. The model will ignore this input if the user did not choose a general port.

Emissions: These emissions data are in the output of the Port Emissions Tool.

Optional Inputs

These inputs are optional for the model to calculate the health impact of port emissions.

Population: Pre-filled with data in the model according to the port and year chosen in the ‘required inputs’ page. Users can change it. The tool is modeling the whole 100*100 km domain to include impacts of secondary PM, so if the user inputs city population instead of domain population, the health impact will be underestimated (assuming the domain is larger than the city).

Ambient concentration: Pre-filled with data from the model according to the port and year chosen in the ‘required inputs’ page. Users can change it.

Incidence rates: Rather than using the model's incidence rate data, users can input their own data. Lung cancer and cardiopulmonary diseases only apply to people with an age of 30 or above. Acute respiratory Infections only apply to children under 5 years old. Unit is deaths/ 100,000 people. Users must input all incidence rates listed on this page if they wish to use their own incidence rate data. If they leave some of them blank, the tool will still use its own underlying data. On the results page, users will be able to see if the model uses its own data or the user’s incidence rate data.

Results

This page shows health impact broken down by different categories. If there are no text under the title ‘Results’ saying that the model is using the user’s incidence rate data, then the model is using its own incidence rate data in the calculations. Each port has three broad areas, on-shore, off-shore and port activities (see Appendix for what emissions are included in each area). Each one of these three areas contains one or more emissions zones. The number of zones in each area for all the ports are detailed below.
